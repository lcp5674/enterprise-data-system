## 1 缓存架构概述

### 1.1 缓存架构目标

缓存架构设计旨在提升系统性能、降低数据库负载、提升用户体验。通过多级缓存策略实现数据的快速访问，通过分布式缓存实现服务间共享，通过缓存预热和失效策略保障数据一致性。

### 2.6 缓存一致性方案

#### 2.6.1 双写一致性

```java
// 方案一：先更新数据库，再删除缓存
@Transactional
public void updateAsset(Asset asset) {
    // 1. 更新数据库
    assetRepository.update(asset);
    
    // 2. 删除缓存（延迟删除，使用消息队列）
    messageQueue.publish(new CacheInvalidateEvent("asset", asset.getId()));
}

// 方案二：使用分布式锁
@Transactional
public void updateAssetWithLock(Asset asset) {
    String lockKey = "lock:asset:" + asset.getId();
    try {
        // 1. 获取分布式锁
        redisTemplate.opsForValue().setIfAbsent(lockKey, "1", 30, TimeUnit.SECONDS);
        
        // 2. 更新数据库
        assetRepository.update(asset);
        
        // 3. 删除缓存
        redisTemplate.delete("asset:" + asset.getId());
    } finally {
        redisTemplate.delete(lockKey);
    }
}
```

#### 2.6.2 异步缓存刷新

```java
// 缓存刷新消息处理
@KafkaListener(topics = "cache-invalidate")
public void handleCacheInvalidate(CacheInvalidateEvent event) {
    String cacheKey = event.getKeyPrefix() + ":" + event.getKeyId();
    redisTemplate.delete(cacheKey);
    
    // 如果需要预热，异步加载新数据
    if (event.isPreload()) {
        CompletableFuture.runAsync(() -> {
            Object data = loadFromDatabase(event);
            redisTemplate.opsForValue().set(cacheKey, data, event.getTtl());
        });
    }
}
```

```java
public class CaffeineConfig {

    @Bean
    public Cache<String, SystemConfig> systemConfigCache() {

            .maximumSize(1000)

            .build();

            .expireAfterWrite(1, TimeUnit.HOURS)
            .recordStats()
            .build();
    }
    
    // 热点资产元数据缓存
    @Bean
    public LoadingCache<String, AssetMetadata> hotAssetCache(

        return Caffeine.newBuilder()

            .expireAfterWrite(5, TimeUnit.MINUTES)

            .build(loader);

    // 用户权限本地缓存

        return Caffeine.newBuilder()
            .maximumSize(10000)
            .expireAfterWrite(30, TimeUnit.MINUTES)
            .expireAfterAccess(10, TimeUnit.MINUTES)
            .recordStats()
            .build();
    }
```

### 3.2 本地缓存更新机制

```java
// 本地缓存与分布式缓存同步
@Component
public class LocalCacheSyncService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    // 监听Redis变更，同步更新本地缓存
    @KafkaListener(topics = "cache-update")
    public void handleCacheUpdate(CacheUpdateEvent event) {
        String localKey = event.getKey();
        
        // 更新Caffeine本地缓存
        if ("system_config".equals(event.getCategory())) {
            SystemConfig config = (SystemConfig) event.getData();
            systemConfigCache.put(localKey, config);
        } else if ("enum_dict".equals(event.getCategory())) {
            List<EnumItem> items = (List<EnumItem>) event.getData();
            enumCache.put(localKey, items);
        }
    }
}
```

## 4 缓存高可用设计

### 4.1 Redis集群故障切换

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Redis故障切换流程                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 故障检测                                                            │
│     • Redis Sentinel 每秒发送PING命令                                   │
│     • 连续3次无响应判定为主观下线                                       │
│     • 半数Sentinel判定为客观下线                                        │
│                                                                         │
│  2. 主从切换                                                            │
│     • Sentinel选举新主节点                                               │
│     • 其他从节点切换到新主                                              │
│     • 通知应用程序新的主节点地址                                        │
│                                                                         │
│  3. 故障恢复                                                            │
│     • 原主节点修复后作为从节点重新加入                                  │
│     • 自动同步新主数据                                                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 缓存降级策略

| 故障场景 | 降级策略 | 影响评估 | 恢复方式 |
|----------|----------|----------|----------|
| 单节点故障 | 自动切换从节点 | 轻微延迟增加 | 自动恢复 |
| 部分节点故障 | 哈希槽重新分配 | 部分缓存miss | 自动恢复 |
| 集群不可用 | 穿透数据库 | 数据库压力增加 | 限流+降级 |
| 网络分区 | 本地缓存兜底 | 功能降级 | 人工介入 |

### 4.3 缓存保护机制

```java
// 熔断器保护
@Component
public class RedisCircuitBreaker {
    
    private CircuitBreaker circuitBreaker;
    
    public Object getWithProtection(String key) {
        try {
            if (circuitBreaker.isOpen()) {
                // 熔断打开，返回空或降级数据
                return getFallbackData(key);
            }
            
            return redisTemplate.opsForValue().get(key);
        } catch (RedisConnectionException e) {
            circuitBreaker.recordFailure();
            return getFallbackData(key);
        }
    }
}

// 限流保护
@Component
public class CacheRateLimiter {
    
    public boolean isAllowed(String key, int maxRequests, int windowSeconds) {
        String rateLimitKey = "ratelimit:" + key;
        Long count = redisTemplate.opsForValue().increment(rateLimitKey);
        
        if (count != null && count == 1) {
            redisTemplate.expire(rateLimitKey, windowSeconds, TimeUnit.SECONDS);
        }
        
        return count != null && count <= maxRequests;
    }
}
```

## 6 最佳实践

### 6.1 缓存使用原则

1. **缓存适合场景**：热点数据访问、计算结果复用、分布式锁、限流计数、会话管理
2. **缓存不适合场景**：频繁变更数据、强一致性要求、敏感隐私数据、超大对象
3. **Key设计原则**：简洁明了、层次分明、避免冲突、控制长度
4. **Value设计原则**：控制大小、选择合适序列化、避免嵌套过深

### 6.2 性能优化建议

1. **批量操作优先**：使用MGET/MSET代替循环GET/SET
2. **Pipeline模式**：减少网络往返次数
3. **连接复用**：使用连接池，避免频繁创建连接
4. **选择合适数据结构**：String用于简单值，Hash用于对象，Set用于去重
5. **避免大Key**：单个Value不超过10MB
6. **合理设置TTL**：避免缓存无限增长

### 6.3 缓存异常处理

```java
@Component
public class CacheService {
    
    private static final Logger logger = LoggerFactory.getLogger(CacheService.class);
    
    public <T> T get(String key, Class<T> type, Supplier<T> loader) {
        try {
            // 1. 尝试从缓存获取
            T value = redisTemplate.opsForValue().get(key);
            if (value != null) {
                return value;
            }
            
            // 2. 缓存未命中，从数据源加载
            value = loader.get();
            
            // 3. 写入缓存
            if (value != null) {
                redisTemplate.opsForValue().set(key, value, 30, TimeUnit.MINUTES);
            }
            
            return value;
        } catch (RedisConnectionException e) {
            // Redis故障，降级到数据库
            logger.warn("Redis连接失败，降级到数据库: {}", key);
            return loader.get();
        } catch (Exception e) {
            logger.error("缓存操作异常: {}", key, e);
            return loader.get();
        }
    }
}
```
