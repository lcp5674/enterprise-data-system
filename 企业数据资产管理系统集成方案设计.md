## 2 CDC数据采集方案

### 2.1 CDC技术选型

| 数据源 | CDC工具 | 说明 |
|--------|---------|------|
| MySQL 5.7+ | Debezium | 开源，支持全量和增量 |
| MySQL 8.0 | Debezium/Canal | 两者均可 |
| PostgreSQL | Debezium | 支持逻辑复制 |
| Oracle | Debezium (Xstream) | 需要Oracle Advanced |
| MongoDB | Debezium | 支持MongoDB Connector |
| SQL Server | Debezium | 支持CDC |

### 2.2 Debezium部署架构

```yaml
# Debezium配置示例
connector:
  name: mysql-connector
  config:
    connector.class: io.debezium.connector.mysql.MySqlConnector
    database.hostname: mysql-source.company.com
    database.port: 3306
    database.user: dam_reader
    database.password: ${CDC_PASSWORD}
    database.server.id: 85744
    database.include.list: crm_db,erp_db
    table.include.list: crm_db.customer,crm_db.order
    schema.history.internal.kafka.bootstrap.servers: kafka:9092
    schema.history.internal.kafka.topic: schema-changes
    topic.prefix: dam
    decimal.handling.mode: double
    time.precision.mode: adaptive
```

#### LDAP/AD集成

```yaml
# LDAP配置
ldap:
  url: ldap://ldap.company.com:389
  base_dn: dc=company,dc=com
  user_search_base: ou=People

  user_attribute_mapping:
    username: uid
    email: mail
```

## 5 Webhook集成

### 5.1 Webhook事件类型

| 事件类型 | 触发条件 | 示例场景 |
|----------|----------|----------|
| ASSET_CREATED | 资产创建完成 | 新表注册 |
| ASSET_UPDATED | 资产信息变更 | 字段修改 |
| ASSET_DELETED | 资产删除 | 表下线 |
| QUALITY_CHECK_PASSED | 质量检测通过 | 数据质量达标 |
| QUALITY_CHECK_FAILED | 质量检测失败 | 发现数据问题 |
| SENSITIVITY_CHANGED | 敏感等级变更 | 安全升级 |
| LINEAGE_UPDATED | 血缘关系更新 | ETL任务变更 |
| PERMISSION_CHANGED | 权限变更 | 访问授权变化 |

### 5.2 Webhook配置

```yaml
webhooks:
  - name: 飞书通知
    url: https://open.feishu.cn/open-apis/bot/v2/hook/xxx
    secret: ${WEBHOOK_SECRET}
    events:
      - QUALITY_CHECK_FAILED
      - SENSITIVITY_CHANGED
    enabled: true
    retry:
      max_attempts: 3
      backoff: exponential
```

### 5.3 签名验证

```python
import hmac
import hashlib
import time

def verify_webhook_signature(payload: str, signature: str, secret: str) -> bool:
    """
    验证Webhook签名
    """
    timestamp = str(int(time.time()))
    message = f"{timestamp}.{payload}"
    expected_signature = hmac.new(
        secret.encode(),
        message.encode(),
        hashlib.sha256
    ).hexdigest()
    
    return hmac.compare_digest(f"v1={expected_signature}", signature)
```

## 7 集成监控

### 7.1 监控指标

| 指标类别 | 指标名称 | 告警阈值 |
|----------|----------|----------|
| CDC延迟 | cdc.lag.seconds | >60秒 |
| CDC错误 | cdc.error.count | >0 |
| Kafka消费延迟 | kafka.consumer.lag | >10000 |
| API调用失败 | api.call.failure.rate | >1% |
| 同步任务失败 | sync.task.failure | >0 |

### 7.2 告警配置

```yaml
alerts:
  - name: CDC延迟告警
    condition: cdc.lag.seconds > 60
    level: WARNING
    channels: [dingtalk, email]
    
  - name: 同步任务失败
    condition: sync.task.failure == true
    level: CRITICAL
    channels: [phone, dingtalk]
```
