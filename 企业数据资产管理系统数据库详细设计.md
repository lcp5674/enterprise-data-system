## 1 设计概述

### 1.1 数据库选型

| 数据库类型 | 用途 | 选型 |
|------------|------|------|
| 关系型数据库 | 核心业务数据 | PostgreSQL 15+ |
| 图数据库 | 血缘关系存储 | Neo4j 5.x |
| 文档数据库 | 非结构化配置 | MongoDB 6.x |
| 时序数据库 | 质量指标存储 | TimescaleDB |
| 搜索引擎 | 全文检索 | Elasticsearch 8.x |
| 缓存数据库 | 会话/热点数据 | Redis 7.x |
| 对象存储 | 文件/附件 | MinIO/S3 |

### 1.2 分库策略

| 库名 | 用途 | 拆分依据 | 数据量预估 |
|------|------|----------|-----------|
| metadata_db | 技术元数据、业务元数据 | 按业务域 | 500GB |
| lineage_db | 血缘关系数据 | 独立库 | 200GB |
| security_db | 权限、安全配置 | 按租户 | 50GB |
| audit_db | 审计日志 | 按时间 | 1TB/月 |
| workflow_db | 工作流、任务 | 按业务域 | 100GB |
| analytics_db | 分析指标、报表 | 按时间 | 200GB |

### 1.3 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| 数据库 | {project}_{module} | dam_asset |
| 表 | {domain}_{entity} | asset_info, user_profile |
| 字段 | snake_case | user_name, create_time |
| 索引 | idx_{table}_{column} | idx_asset_owner_id |
| 主键 | id (UUID) | - |
| 外键 | {table}_id | department_id |
| 软删除 | deleted_at | - |

## 2 核心表结构设计

### 2.1 用户与认证模块

#### 用户信息表 (sys_user)

```sql
CREATE TABLE sys_user (
    id              VARCHAR(32) PRIMARY KEY DEFAULT gen_random_uuid()::text,
    username        VARCHAR(50) NOT NULL UNIQUE,
    password        VARCHAR(255) NOT NULL,
    nickname        VARCHAR(100),
    email           VARCHAR(100) NOT NULL UNIQUE,
    phone           VARCHAR(20),
    employee_no     VARCHAR(50),
    avatar_url      VARCHAR(500),
    department_id   VARCHAR(32),
    status          SMALLINT DEFAULT 1,          -- 1:启用 0:禁用
    user_type       SMALLINT DEFAULT 1,          -- 1:内部用户 2:外部用户
    last_login_time TIMESTAMP,
    last_login_ip   VARCHAR(50),
    password_expire_time TIMESTAMP,
    login_fail_count SMALLINT DEFAULT 0,
    is_deleted      SMALLINT DEFAULT 0,
    created_by      VARCHAR(32),
    created_time    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by      VARCHAR(32),
    updated_time    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    CONSTRAINT uk_username UNIQUE (username),
    CONSTRAINT uk_email UNIQUE (email)
);

CREATE INDEX idx_sys_user_department ON sys_user(department_id);
CREATE INDEX idx_sys_user_status ON sys_user(status);
CREATE INDEX idx_sys_user_created_time ON sys_user(created_time);
```

### 2.2 资产管理模块

```sql

    table_name          VARCHAR(200),

    business_domain     VARCHAR(100),                   -- 业务域

    status              VARCHAR(20) DEFAULT 'DRAFT',     -- DRAFT/PENDING/APPROVED/REJECTED/OBSOLETE
    row_count           BIGINT,
    storage_size        VARCHAR(50),
    record_version      BIGINT DEFAULT 1,              -- 版本号
    source_system       VARCHAR(100),                  -- 来源系统
    source_table        VARCHAR(200),                  -- 来源表
    encryption_type     VARCHAR(50),                   -- 加密类型
    retention_days      INTEGER,                        -- 保留天数
    tags                JSONB DEFAULT '[]',            -- 标签数组
    extra_info          JSONB DEFAULT '{}',            -- 扩展信息
    is_deleted          SMALLINT DEFAULT 0,
    created_by          VARCHAR(32),
    created_time        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by          VARCHAR(32),
    updated_time        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_by          VARCHAR(32),
    deleted_time        TIMESTAMP

CREATE INDEX idx_asset_name ON asset_info(name);
CREATE INDEX idx_asset_type ON asset_info(asset_type);
CREATE INDEX idx_asset_domain ON asset_info(business_domain);
CREATE INDEX idx_asset_sensitivity ON asset_info(sensitivity_level);
```

## 3 Neo4j图模型设计

### 3.1 节点定义

```cypher
// 数据资产节点
(:Asset {
    assetId: String,
    name: String,
    type: String,        // TABLE/FILE/API
    sensitivityLevel: String,
    businessDomain: String,
    ownerId: String,
    createTime: DateTime,
    updateTime: DateTime
})

// 字段节点
(:Field {
    fieldId: String,
    name: String,
    dataType: String,
    isPrimaryKey: Boolean,
    sensitivityLevel: String
})

// 数据源节点
(:Datasource {
    datasourceId: String,
    name: String,
    type: String
})

// 任务节点
(:Task {
    taskId: String,
    name: String,
    type: String,       // ETL/SPARK/FLINK
    schedule: String
})

// 人员节点
(:Person {
    userId: String,
    name: String,
    role: String
})
```

### 3.2 关系定义

```cypher
// 血缘关系
(:Asset)-[:LINEAGE {
    type: String,        // ETL/SQL/MANUAL
    transformSql: String,
    taskName: String,
    confidence: Float,
    createTime: DateTime
}]->(:Asset)

// 字段级血缘
(:Field)-[:FIELD_LINEAGE {
    transform: String,
    createTime: DateTime
}]->(:Field)

// 属于关系
(:Asset)-[:BELONGS_TO]->(:Datasource)
(:Field)-[:BELONGS_TO]->(:Asset)

// 负责人关系
(:Asset)-[:OWNED_BY]->(:Person)

// 依赖关系
(:Task)-[:EXTRACTS_FROM]->(:Asset)
(:Task)-[:LOADS_TO]->(:Asset)
```

### 3.3 常用查询

```cypher
// 查询表级血缘
MATCH (a:Asset {assetId: $assetId})-[r:LINEAGE*1..3]->(b:Asset)
RETURN a, r, b

// 查询字段级血缘
MATCH (f1:Field {fieldId: $fieldId})-[r:FIELD_LINEAGE*1..3]->(f2:Field)
RETURN f1, r, f2

// 影响分析
MATCH (a:Asset {assetId: $assetId})-[*1..5]->(affected:Asset)
WHERE NOT (a)<-[:LINEAGE*]-(affected)
RETURN DISTINCT affected

// 追溯分析
MATCH (source:Asset)-[*1..5]->(a:Asset {assetId: $assetId})
WHERE NOT (source)<-[:LINEAGE*]-(a)
RETURN DISTINCT source
```
