## 1 设计概述

### 1.1 数据库选型

| 数据库类型 | 用途 | 选型 | 数据量预估 |
|------------|------|------|-----------|
| 关系型数据库 | 核心业务数据 | PostgreSQL 15+ | 10TB |
| 图数据库 | 血缘关系存储 | Neo4j 5.x | 200GB |
| 文档数据库 | 非结构化配置 | MongoDB 6.x | 50GB |
| 时序数据库 | 质量指标存储 | TimescaleDB | 100GB |
| 搜索引擎 | 全文检索 | Elasticsearch 8.x | 200GB |
| 缓存数据库 | 会话/热点数据 | Redis 7.x Cluster | 50GB |
| 对象存储 | 文件/附件 | MinIO/S3 | 500GB |
| 向量数据库 | AI向量存储 | Milvus 2.x | 100GB |

### 1.2 分库策略

| 库名 | 用途 | 拆分依据 | 数据量预估 |
|------|------|----------|-----------|
| dam_core | 核心业务数据 | - | 500GB |
| dam_metadata | 元数据存储 | 按业务域 | 300GB |
| dam_lineage | 血缘关系数据 | 独立库 | 200GB |
| dam_security | 权限、安全配置 | 按租户 | 50GB |
| dam_audit | 审计日志 | 按时间 | 1TB/月 |
| dam_workflow | 工作流、任务 | 按业务域 | 100GB |
| dam_analytics | 分析指标、报表 | 按时间 | 200GB |
| dam_knowledge | 知识图谱 | 独立库 | 100GB |
| dam_iot | IoT时序数据 | 独立库 | 500GB |

### 1.3 分表策略

| 表名 | 分表键 | 分表数 | 路由规则 |
|------|--------|--------|----------|
| asset_info | tenant_id | 64 | hash(tenant_id) % 64 |
| metadata_field | asset_id | 256 | hash(asset_id) % 256 |
| lineage_node | batch_id | 128 | hash(batch_id) % 128 |
| audit_log | created_time | 按月 | PARTITION BY RANGE |
| quality_result | check_id | 256 | hash(check_id) % 256 |
| access_log | access_time | 按月 | PARTITION BY RANGE |
| notification | recipient_id | 64 | hash(recipient_id) % 64 |

### 1.4 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| 数据库 | {project}_{module} | dam_asset, dam_metadata |
| 表 | {domain}_{entity} | asset_info, user_profile |
| 字段 | snake_case | user_name, create_time |
| 索引 | idx_{table}_{column} | idx_asset_owner_id |
| 主键 | id (UUID) | - |
| 外键 | {table}_id | department_id |
| 软删除 | deleted_at | - |
| 时间戳 | _time/_at | create_time, updated_at |

### 1.5 通用字段

所有表包含以下通用字段：

```sql
-- 审计字段
id              VARCHAR(32) PRIMARY KEY  -- UUID
created_by      VARCHAR(32)              -- 创建人
created_time    TIMESTAMP DEFAULT NOW() -- 创建时间
updated_by      VARCHAR(32)              -- 更新人
updated_time    TIMESTAMP DEFAULT NOW() -- 更新时间

-- 软删除字段
is_deleted      SMALLINT DEFAULT 0      -- 是否删除
deleted_by      VARCHAR(32)              -- 删除人
deleted_time     TIMESTAMP               -- 删除时间
```

CREATE TABLE sys_user (

    username            VARCHAR(50) NOT NULL UNIQUE,

    source_type         VARCHAR(20),                 -- LOCAL/LDAP/OAUTH2
    source_id           VARCHAR(100),                 -- 第三方用户ID

    created_time        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by         VARCHAR(32),

    CONSTRAINT uk_sys_user_username UNIQUE (username),
    CONSTRAINT uk_sys_user_email UNIQUE (email),

### 21.3 常用查询

```cypher
// 查询表级血缘
MATCH (a:Asset {assetId: $assetId})
CALL {
    WITH a
    MATCH (a)-[r:LINEAGE*1..5]->(b:Asset)
    RETURN a, r, b
    UNION
    WITH a
    MATCH (b:Asset)-[r:LINEAGE*1..5]->(a)
    RETURN b as a, r, a as b
}
RETURN DISTINCT a, r, b

// 查询字段级血缘
MATCH (f1:Field {fieldId: $fieldId})-[r:FIELD_LINEAGE*1..3]->(f2:Field)
RETURN f1, r, f2

// 影响分析
MATCH (a:Asset {assetId: $assetId})-[*1..5]->(affected:Asset)
WHERE NOT (a)<-[:LINEAGE*]-(affected)
RETURN DISTINCT affected

// 追溯分析
MATCH (source:Asset)-[*1..5]->(a:Asset {assetId: $assetId})
WHERE NOT (source)<-[:LINEAGE*]-(a)
RETURN DISTINCT source

// 环路检测
MATCH path = (a:Asset)-[:LINEAGE*]->(a)
WHERE length(path) > 0
RETURN path

// 批量血缘查询（性能优化）
MATCH (a:Asset)-[r:LINEAGE]->(b:Asset)
WHERE a.assetId IN $assetIds
RETURN a.assetId as source, collect({target: b.assetId, type: r.type}) as relations
```
