## 1 引言

### 1.1 文档目的

本文档定义企业数据资产管理系统的功能需求、非功能需求和验收标准，作为系统设计、开发和测试的依据。文档涵盖系统的核心功能模块、业务流程、用户角色、数据需求和技术约束，为项目团队提供明确的需求指导。

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| 数据资产 | 企业拥有的、具有业务价值的数据资源，包括数据表、文件、指标、接口等 |
| 元数据 | 描述数据的数据，包括技术元数据、业务元数据和管理元数据 |
| 血缘关系 | 数据在采集、传输、加工过程中形成的来源与去向关系 |
| 数据目录 | 按业务域或主题分类组织的数据资产清单 |
| 资产标签 | 描述数据资产特征的可自定义关键词 |
| 安全等级 | 根据数据敏感程度划分的保护级别 |
| 数据治理 | 对数据资产进行管理和控制的一系列活动和流程 |
| 数据价值 | 数据资产对企业的业务价值和经济价值 |
| 边缘计算 | 在靠近数据源的边缘设备上进行数据处理和分析 |
| 数据伦理 | 数据使用过程中应遵循的道德准则和规范 |

## 2 功能性需求

### 2.1 基础支撑域

#### 2.1.1 登录注册模块

**功能描述**：提供用户身份认证和账号管理能力。

**功能点**：
- 用户注册：支持手机号、邮箱注册，支持企业邮箱自动关联企业信息
- 用户登录：支持账号密码登录、手机验证码登录、第三方登录（企业微信、钉钉、LDAP）
- 忘记密码：支持通过手机或邮箱重置密码
- 账号管理：支持个人信息修改、密码修改、绑定手机号/邮箱

**用户场景**：
- 新用户注册并登录系统
- 现有用户登录系统进行数据资产管理操作
- 用户忘记密码后重置密码

#### 2.1.2 用户管理模块

**功能描述**：提供企业组织架构、用户账号和用户配置的集中管理能力。

**功能点**：
- 组织架构管理：支持企业组织架构的创建、修改、删除，支持多级组织树结构
- 用户管理：支持用户的创建、修改、删除、启用/禁用，支持批量操作
- 用户导入/导出：支持从Excel导入用户，支持用户信息导出
- 用户配置：支持用户个性化设置，如语言偏好、主题设置、通知偏好

**用户场景**：
- 管理员创建新用户账号并分配角色
- 管理员调整企业组织架构
- 管理员批量导入新员工用户信息

#### 2.1.3 权限管理模块

**功能描述**：实现系统的功能权限和数据权限的精细化管控。

**功能点**：
- 角色管理：支持角色的创建、修改、删除，支持预设角色和自定义角色
- 权限分配：支持为角色分配功能权限和数据权限，支持权限的批量分配
- 权限审批：支持权限申请、审批流程，支持临时权限和永久权限
- 权限审计：支持权限分配历史查询，支持权限使用情况分析

**用户场景**：
- 管理员创建新角色并分配权限
- 用户申请超出其角色的临时权限
- 管理员审计权限分配情况

#### 2.1.4 系统配置模块

**功能描述**：提供系统运行参数、个性化设置和运维配置的管理能力。

**功能点**：
- 系统参数配置：支持全局参数设置，如系统名称、Logo、通知模板等
- 数据源配置：支持各类数据源的连接配置，包括数据库、文件系统、API等
- 第三方集成配置：支持与外部系统的集成配置，如SSO、消息通知等
- 字典代码表配置：支持系统内置字典的维护和管理
- 运维参数配置：支持系统性能参数、日志级别等运维相关配置
- 连通性测试中心：支持数据源连接测试、API连通性测试等

**用户场景**：
- 管理员配置系统全局参数
- 技术人员配置新的数据源连接
- 运维人员调整系统性能参数

#### 2.1.5 通知服务模块

**功能描述**：提供多渠道消息推送和通知管理能力。

**功能点**：
- 通知渠道管理：支持站内信、邮件、短信、企业微信、钉钉等多种通知渠道
- 通知模板管理：支持通知内容模板的创建、修改、删除
- 通知发送：支持手动发送通知和系统自动触发通知
- 通知管理：支持通知历史查询、通知状态跟踪、通知撤回

**用户场景**：
- 系统自动发送数据质量异常通知
- 管理员手动发送系统维护通知
- 用户查看历史通知记录

#### 2.1.6 审计日志模块

**功能描述**：完整记录用户在系统中的所有操作行为。

**功能点**：
- 操作日志：记录用户的登录、退出、功能操作等行为
- 安全日志：记录安全相关事件，如权限变更、敏感操作等
- 系统日志：记录系统运行状态、异常事件等
- 日志查询：支持按时间、用户、操作类型、关键词等条件查询日志
- 日志导出：支持日志的导出和归档

**用户场景**：
- 审计人员查询用户操作记录
- 安全管理员分析安全事件
- 运维人员排查系统异常

### 2.2 资产发现域

#### 2.2.1 数据地图模块

**功能描述**：提供企业数据资产的全局视图和探索能力。

**功能点**：
- 资产概览：展示企业数据资产的总量、分布、分类等统计信息
- 资产浏览：支持按业务域、数据类型、部门等维度浏览资产
- 资产搜索：支持关键字搜索、高级搜索、过滤筛选
- 资产详情：展示单个资产的详细信息，包括基本信息、技术属性、业务属性等
- 资产关系：展示资产之间的关联关系，如血缘关系、引用关系等

**用户场景**：
- 用户通过数据地图浏览企业数据资产
- 用户搜索特定的数据资产
- 用户查看资产的详细信息和关联关系

#### 2.2.2 资产搜索模块

**功能描述**：提供强大的数据资产检索能力。

**功能点**：
- 全文搜索：支持对资产名称、描述、标签等字段的全文检索
- 高级搜索：支持按资产类型、安全等级、创建时间等条件进行组合搜索
- 标签搜索：支持按资产标签进行搜索
- 搜索历史：记录用户的搜索历史，支持搜索历史的管理
- 搜索建议：根据用户输入提供搜索建议，提升搜索体验

**用户场景**：
- 用户通过关键字搜索数据资产
- 用户使用高级搜索条件精确定位资产
- 用户通过标签快速筛选资产

#### 2.2.3 数据目录模块

**功能描述**：提供数据资产的多维度分类组织能力。

**功能点**：
- 目录结构管理：支持创建、修改、删除目录，支持多级目录结构
- 资产分类：支持将资产归类到不同目录，支持批量操作
- 目录权限：支持为目录设置访问权限，控制目录的可见性
- 目录导出：支持目录结构和资产清单的导出

**用户场景**：
- 管理员创建数据资产分类目录
- 用户将资产归类到相应目录
- 用户通过目录浏览资产

#### 2.2.4 资产详情模块

**功能描述**：呈现单个数据资产的完整信息。

**功能点**：
- 基本信息：展示资产名称、类型、来源、创建时间等基本信息
- 技术属性：展示资产的技术特征，如数据结构、存储位置、访问方式等
- 业务属性：展示资产的业务含义、使用场景、关联业务流程等
- 使用信息：展示资产的使用情况、访问频率、使用热度等
- 关联资产：展示与当前资产相关的其他资产，如上游/下游资产、引用资产等

**用户场景**：
- 用户查看资产的详细信息
- 用户了解资产的技术和业务属性
- 用户分析资产的使用情况和关联关系

### 2.3 资产治理域

#### 2.3.1 数据血缘模块

**功能描述**：描述数据在采集、传输、加工、转换、加载等处理过程中的来源与去向关系。

**功能点**：
- 血缘可视化：以有向无环图展示血缘关系，支持缩放、拖拽、展开/收起
- 血缘追踪：支持上游追溯、下游追踪、全链分析
- 影响分析：分析数据变更对下游资产的影响范围
- 血缘对比：支持不同时间点血缘关系的对比分析
- 血缘导出：支持血缘关系的导出和分享

**用户场景**：
- 用户分析数据资产的来源和去向
- 用户评估数据变更的影响范围
- 用户导出血缘关系用于文档或报告

#### 2.3.2 数据模型模块

**功能描述**：用于管理和维护企业的数据仓库模型。

**功能点**：
- 模型设计：支持概念模型、逻辑模型、物理模型的多层建模
- 模型版本：支持模型的版本管理，包括版本创建、对比、回滚
- 模型导入/导出：支持模型的导入和导出，支持多种模型格式
- 模型评审：支持模型的评审流程，包括评审任务分配、评审意见收集、评审结果跟踪

**用户场景**：
- 数据建模人员设计和维护数据模型
- 团队成员评审数据模型设计
- 技术人员导入/导出数据模型

#### 2.3.3 数据标准模块

**功能描述**：提供业务术语字典、数据命名规范和代码值标准的管理能力。

**功能点**：
- 术语管理：支持业务术语的创建、修改、删除，支持术语的层级关系
- 命名规范：定义数据资产的命名规则，支持规则的验证和应用
- 代码值标准：管理代码值的标准定义，支持代码值的版本管理
- 标准应用：将标准应用到数据资产，支持标准合规性检查

**用户场景**：
- 数据标准管理员定义和维护数据标准
- 用户查询业务术语和代码值标准
- 系统自动检查数据资产的标准合规性

#### 2.3.4 指标管理模块

**功能描述**：支持指标的统一定义、血缘追溯和口径统一。

**功能点**：
- 指标定义：支持指标的创建、修改、删除，包括指标名称、定义、口径、计算方法等
- 指标血缘：追踪指标的数据源和计算过程，确保指标口径的可追溯性
- 指标版本：支持指标的版本管理，包括版本创建、对比、回滚
- 指标目录：支持指标的分类组织，便于查找和管理
- 指标分析：支持指标的趋势分析、对比分析、异常分析

**用户场景**：
- 业务人员定义和维护业务指标
- 分析师查询指标的定义和计算口径
- 管理层分析指标的趋势和异常

### 2.4 安全合规域

#### 2.4.1 分类分级模块

**功能描述**：建立数据资产的分类体系和分级标准。

**功能点**：
- 分类体系：定义数据资产的分类标准，支持多级分类结构
- 分级标准：定义数据资产的安全等级，如公开、内部、保密、机密等
- 自动分类：基于规则自动识别和分类数据资产
- 手动分类：支持用户手动调整数据资产的分类和分级
- 分类统计：统计不同分类和分级的数据资产数量和分布

**用户场景**：
- 安全管理员定义数据分类分级标准
- 系统自动对数据资产进行分类分级
- 用户手动调整数据资产的分类分级

#### 2.4.2 权限控制模块

**功能描述**：提供基于角色和属性细粒度的权限控制。

**功能点**：
- 功能权限：控制用户对功能模块的访问和操作权限
- 数据权限：控制用户对数据资产的访问权限，如行级权限、列级权限
- 权限申请：支持用户申请超出其角色的临时权限
- 权限审批：支持权限申请的审批流程，包括多级审批
- 权限审计：审计权限的分配和使用情况

**用户场景**：
- 管理员为用户分配功能权限和数据权限
- 用户申请临时权限访问特定数据
- 安全管理员审计权限分配情况

#### 2.4.3 脱敏管理模块

**功能描述**：定义敏感数据的脱敏规则。

**功能点**：
- 脱敏规则：定义不同类型敏感数据的脱敏规则，如手机号、身份证号、银行卡号等
- 脱敏策略：设置脱敏策略，包括脱敏范围、脱敏方式、脱敏级别
- 动态脱敏：在数据查询和展示时实时脱敏
- 静态脱敏：对存储的数据进行脱敏处理
- 脱敏审计：审计脱敏规则的应用情况

**用户场景**：
- 安全管理员定义敏感数据脱敏规则
- 系统在数据展示时自动脱敏敏感信息
- 安全管理员审计脱敏规则的应用情况

#### 2.4.4 合规审计模块

**功能描述**：提供合规检查清单、合规报告生成和隐私影响评估能力。

**功能点**：
- 合规检查：根据法规要求进行合规检查，如《数据安全法》《个人信息保护法》等
- 合规报告：自动生成合规报告，包括合规状态、问题清单、改进建议
- 隐私影响评估：对数据处理活动进行隐私影响评估
- 合规任务：管理合规相关任务，包括任务分配、执行、跟踪
- 合规历史：记录合规检查和评估的历史结果

**用户场景**：
- 合规专员进行合规检查和评估
- 管理层查看合规报告
- 系统自动生成合规任务并跟踪执行情况

### 2.5 质量保障域

#### 2.5.1 质量规则模块

**功能描述**：定义数据质量检测规则。

**功能点**：
- 规则定义：支持创建、修改、删除质量检测规则，包括空值检测、唯一性检测、值域检测、格式检测、关联一致性检测等
- 规则模板：提供常用规则模板，便于快速创建规则
- 规则分组：支持规则的分类和分组管理
- 规则测试：支持规则的测试和验证

**用户场景**：
- 质量管理员定义数据质量检测规则
- 用户使用规则模板快速创建规则
- 质量管理员测试和验证规则

#### 2.5.2 质量检测模块

**功能描述**：按照规则执行质量检测。

**功能点**：
- 检测任务：创建和管理质量检测任务，包括任务配置、执行计划、执行状态
- 定时检测：支持设置定时检测任务，自动执行质量检测
- 手动检测：支持手动触发质量检测
- 检测结果：展示检测结果，包括通过、警告、失败等状态
- 问题识别：自动识别数据质量问题，生成问题清单

**用户场景**：
- 质量管理员配置定时质量检测任务
- 用户手动触发质量检测
- 质量管理员查看检测结果和问题清单

#### 2.5.3 问题管理模块

**功能描述**：跟踪发现的质量问题。

**功能点**：
- 问题记录：记录质量检测发现的问题，包括问题描述、严重程度、影响范围
- 问题分配：将问题分配给相关责任人
- 问题处理：支持问题的处理、验证、关闭等流程

## 4 数据需求

### 4.1 数据模型

**核心数据实体**：
- 数据资产：描述企业的数据资产，包括基本信息、技术属性、业务属性等
- 数据源：描述数据资产的来源，包括数据库、文件系统、API等
- 数据分类：描述数据资产的分类体系
- 数据血缘：描述数据资产之间的血缘关系
- 数据质量：描述数据资产的质量状况
- 用户：描述系统用户，包括基本信息、角色、权限等
- 任务：描述系统中的各类任务，如治理任务、质量检测任务等
- 报告：描述系统生成的各类报告，如质量报告、合规报告等

**数据关系**：
- 数据资产与数据源：多对一关系
- 数据资产与数据分类：多对多关系
- 数据资产与数据血缘：一对多关系
- 用户与角色：多对多关系
- 角色与权限：多对多关系

### 4.2 数据存储

**存储策略**：
- 关系型数据：存储在关系型数据库中，如MySQL、PostgreSQL
- 图数据：存储在图数据库中，如Neo4j
- 文档数据：存储在文档数据库中，如MongoDB
- 时序数据：存储在时序数据库中，如TimescaleDB、InfluxDB
- 分析数据：存储在分析型数据库中，如ClickHouse
- 缓存数据：存储在缓存系统中，如Redis
- 非结构化数据：存储在对象存储中，如MinIO、S3

**存储架构**：
- 主从架构：确保数据高可用性
- 读写分离：优化数据库性能
- 分库分表：应对数据量增长
- 数据分片：提高查询性能

### 4.3 数据集成

**数据源集成**：
- 关系型数据库：MySQL、PostgreSQL、Oracle、SQL Server等
- 大数据组件：Hive、HBase、Kafka、Flink等
- 云数据仓库：MaxCompute、Hologres、Snowflake、BigQuery等
- 文件存储：S3、OSS、HDFS、本地文件系统等
- API数据源：REST API、GraphQL等

**集成方式**：
- 连接器：为每种数据源提供专用连接器
- CDC：变更数据捕获，实现实时数据同步
- ETL：抽取、转换、加载，实现批量数据集成
- API调用：通过API接口获取数据

|------|------|----------|----------|----------|
| 一次性迁移 | 全量导出导入 | 数据量<100GB | 中 | <24小时 |
| 分批迁移 | 按时间/业务分批 | 数据量100GB-1TB | 低 | 1-7天 |
| 增量同步 | 双写+CDC | 数据量>1TB | 低 | 持续 |
| 同步双跑 | 新旧系统并行 | 核心业务 | 极低 | 1-4周 |

### 9.4 数据校验规则

| 校验类型 | 校验内容 | 允许偏差 | 校验方式 |
|----------|----------|----------|----------|
| 数量校验 | 记录总数 | 0 | SELECT COUNT(*) |
| 抽样校验 | 随机抽样1000条 | 0 | 逐字段比对 |
| 汇总校验 | 金额、数量合计 | <0.01% | SUM聚合比对 |
| 格式校验 | 日期、编码格式 | 0 | 正则表达式 |
| 业务规则校验 | 关联关系完整性 | <0.1% | 完整性查询 |
| 非空校验 | 必填字段 | 0 | IS NOT NULL |

### 9.5 回滚方案

| 场景 | 触发条件 | 回滚策略 | 回滚时间 | 责任人 |
|------|----------|----------|----------|--------|
| 迁移过程异常 | 工具报错/网络中断 | 暂停迁移，保留增量数据 | <5分钟 | 迁移工程师 |
| 校验发现问题 | 数据不一致>阈值 | 重新执行问题批次 | <30分钟 | 迁移工程师 |
| 上线后发现严重问题 | 业务故障/数据丢失 | 切换回旧系统，数据回写 | <2小时 | 运维负责人 |
| 灰度发布失败 | 灰度指标未达标 | 回滚至发布前版本 | <15分钟 | DevOps |
