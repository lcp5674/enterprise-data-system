### 1.2 协议选型决策矩阵

| 场景 | 推荐协议 | 备选协议 | 说明 |
|------|----------|----------|------|
| 外部API暴露 | REST | GraphQL | 通用性强、易于集成 |
| 服务间同步调用 | gRPC | REST | 高性能、低延迟 |
| 复杂查询/搜索 | REST | GraphQL | 灵活筛选、聚合 |
| 实时数据推送 | WebSocket | SSE | 双向/单向推送 |
| 大文件传输 | gRPC Streaming | REST分片 | 支持流式处理 |
| 事件驱动通信 | Kafka | RabbitMQ | 高吞吐、持久化 |
| 任务异步处理 | RabbitMQ | Kafka | 灵活路由、优先级 |
| 流式计算 | Kafka | Flink | 消息流处理 |

## 2 REST API设计规范

### 2.1 API设计原则

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          REST API设计原则                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 资源导向（Resource-Oriented）                                        │
│     • 使用名词而非动词: /assets 而不是 /getAssets                        │
│     • 使用复数形式: /assets 而不是 /asset                               │
│     • 使用嵌套表示关系: /assets/{id}/columns                            │
│                                                                         │
│  2. HTTP方法语义化                                                       │
│     • GET - 查询资源（幂等、安全）                                       │
│     • POST - 创建资源                                                   │
│     • PUT - 全量更新资源（幂等）                                         │
│     • PATCH - 部分更新资源                                              │
│     • DELETE - 删除资源（幂等）                                         │
│                                                                         │
│  3. 状态码标准化                                                         │
│     • 2xx - 成功                                                        │
│     • 4xx - 客户端错误                                                  │
│     • 5xx - 服务端错误                                                  │
│                                                                         │
│  4. 版本化管理                                                           │
│     • URL路径版本: /api/v1/assets                                       │
│     • Header版本: Accept: application/vnd.api+json;version=1            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3 gRPC协议设计

### 3.1 gRPC适用场景

gRPC适用于以下场景：
- **高性能内部通信**：服务间低延迟调用
- **流式数据处理**：血缘分析、质量检测等大数据量场景
- **双向通信**：实时推送、协作编辑
- **多语言支持**：跨语言服务调用

### 3.2 Proto定义规范

```protobuf
// 用户管理服务Proto定义
syntax = "proto3";

package com.enterprise.dataplatform.asset;

// 选项配置
option java_multiple_files = true;
option java_package = "com.enterprise.dataplatform.asset.grpc";
option java_outer_classname = "AssetServiceProto";
option objc_class_prefix = "EAP";

// 导入公共定义
import "google/protobuf/timestamp.proto";
import "common.proto";

// 资产服务定义
service AssetService {
  // 简单RPC
  rpc GetAsset(GetAssetRequest) returns (GetAssetResponse);
  
  // 服务端流式RPC - 获取血缘关系（支持大量节点）
  rpc StreamLineage(StreamLineageRequest) returns (stream LineageNode);
  
  // 客户端流式RPC - 批量创建资产
  rpc BatchCreateAssets(stream CreateAssetRequest) returns (BatchCreateAssetsResponse);
  
  // 双向流式RPC - 实时数据同步
  rpc SyncAssets(stream SyncAssetRequest) returns (stream SyncAssetResponse);
}

// 通用请求/响应消息
message GetAssetRequest {
  string asset_id = 1;
  repeated string fields = 2;  // 指定返回字段
}

message GetAssetResponse {
  Asset asset = 1;
}

message StreamLineageRequest {
  string asset_id = 1;
  int32 max_depth = 2;         // 最大深度
  Direction direction = 3;     // 方向：UPSTREAM/DOWNSTREAM/BOTH
  bool include_column_level = 4; // 包含字段级血缘
}

enum Direction {
  UPSTREAM = 0;
  DOWNSTREAM = 1;
  BOTH = 2;
}

message LineageNode {
  string asset_id = 1;
  string name = 2;
  string type = 3;
  repeated LineageEdge upstream = 4;
  repeated LineageEdge downstream = 5;
}

message LineageEdge {
  string target_asset_id = 1;
  string transformation = 2;
}

// 资产消息定义
message Asset {
  string id = 1;
  string guid = 2;
  string name = 3;
  string type = 4;
  string source_id = 5;
  string database = 6;
  string schema = 7;
  string owner_id = 8;
  string owner_name = 9;
  string security_level = 10;
  repeated string tags = 11;
  google.protobuf.Timestamp created_at = 12;
  google.protobuf.Timestamp updated_at = 13;
}
```

### 3.3 gRPC与REST对比

| 特性 | gRPC | REST |
|------|------|------|
| 传输协议 | HTTP/2 | HTTP/1.1/2 |
| 序列化 | Protocol Buffers | JSON |
| 消息大小 | 小（二进制） | 大（文本） |
| 性能 | 高 | 中等 |
| 代码生成 | 原生支持 | 需Swagger |
| 浏览器支持 | 需grpc-web | 原生支持 |
| 流式支持 | 原生双向流 | SSE/WebSocket |
| 易读性 | 低（二进制） | 高（JSON） |
| 适用场景 | 内部服务 | 外部API |

### 5.2 负载均衡策略

| 策略 | 原理 | 适用场景 | 配置 |
|------|------|----------|------|
| Round Robin | 轮询分发 | 无状态服务 | 默认 |
| Weighted Round Robin | 加权轮询 | 异构环境 | 按能力分配权重 |
| Random | 随机选择 | 无状态服务 | 简单场景 |
| Least Connections | 最少连接 | 长连接服务 | 连接管理 |
| Consistent Hash | 一致性哈希 | 有状态调用 | 缓存友好 |
| Response Time | 响应时间加权 | 性能敏感 | 自适应 |

### 5.3 熔断器配置

```java
// 熔断器配置示例
@Configuration
public class CircuitBreakerConfig {
    
    @Bean
    public CircuitBreakerFactory circuitBreakerFactory() {
        CircuitBreakerConfig config = CircuitBreakerConfig.custom()
            // 熔断器开启条件
            .failureRateThreshold(50)           // 失败率阈值50%
            .slowCallRateThreshold(80)         // 慢调用率阈值80%
            .slowCallDurationThreshold(Duration.ofSeconds(5))  // 慢调用阈值5秒
            
            // 熔断器时间窗口
            .slidingWindowType(SlidingWindowType.COUNT_BASED)
            .slidingWindowSize(10)             // 滑动窗口大小10次
            
            // 熔断恢复
            .minimumNumberOfCalls(5)           // 最小调用数
            .waitDurationInOpenState(Duration.ofSeconds(60))  // 熔断开启持续60秒
            
            // 允许失败重试
            .permittedNumberOfCallsInHalfOpenState(3)  // 半开状态允许3次调用
            
            .build();
        
        return new SpringCloudCircuitBreaker仿制工厂(config);
    }
}
```

### 6.2 重试策略

```yaml
# 重试配置规范
retry:
  # 全局重试策略
  global:
    maxAttempts: 3
    initialInterval: 1000      # 初始间隔 1秒
    multiplier: 2.0           # 间隔倍增
    maxInterval: 10000        # 最大间隔 10秒
    retryOnException: true
    retryOnResponse: true
    
  # 按HTTP方法重试
  byMethod:
    GET:
      maxAttempts: 2
      retryOn: [500, 502, 503, 504]  # 仅重试网关错误
    POST:
      maxAttempts: 3
      retryOn: [500, 503]
      retryOnException: true
    PUT:
      maxAttempts: 3
      retryOn: [500, 503]
      retryOnException: true
    DELETE:
      maxAttempts: 2
      retryOn: [500, 503]
  
  # 排除不重试的场景
  exclude:
    - 400  # Bad Request
    - 401  # Unauthorized
    - 403  # Forbidden
    - 404  # Not Found
    - 409  # Conflict
```

### 8.2 性能优化建议

1. **连接池复用**：使用HTTP/2或连接池减少连接建立开销
2. **批量操作**：聚合多个小请求为批量请求
3. **压缩传输**：大响应启用gzip压缩
4. **异步非阻塞**：使用响应式编程提升吞吐量
5. **本地缓存**：减少重复请求
6. **流式处理**：大文件使用流式传输
