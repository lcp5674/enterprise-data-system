## 🔧 整改计划

### P0 - 紧急整改（1-2天）

#### 1. JWT密钥管理优化 ✅

**问题**: 硬编码 `edams-jwt-secret-key-2026`

**整改方案**:

1. **JwtProperties.java** - 移除硬编码默认值
```java
// 修改前
private String secret = "edams-jwt-secret-key-2026";

// 修改后 - 必须从环境变量读取
private String secret = System.getenv("JWT_SECRET");
```

2. **application.yml** - 使用环境变量占位符
```yaml
jwt:
  secret: ${JWT_SECRET}  # 强制要求环境变量
```

3. **application-prod.yml** - 生产环境明确标注
```yaml
jwt:
  secret: ${JWT_SECRET}  # 必须在K8s Secret或环境变量中设置
```

#### 2. 数据库密码管理优化 ✅

**问题**: 13个服务使用默认数据库密码

**整改方案**:

1. 所有服务application.yml改为:
```yaml
spring:
  datasource:
    password: ${DB_PASSWORD}
```

2. 创建 `infrastructure/k8s/secrets/db-secrets.yaml`:
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: edams-db-secrets
type: Opaque
data:
  # base64编码的密码
  DB_PASSWORD: <base64-encoded-password>
```

### P1 - 重要整改（3-5天）

#### 3. 监控告警完善 ✅

**当前状态**: 有SkyWalking配置，但告警规则不完善

**整改方案**:

1. 创建 `monitoring/prometheus-alerts.yml`:
```yaml
groups:
  - name: edams-alerts
    rules:
      - alert: ServiceDown
        expr: up{job="edams-*"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Service {{ $labels.instance }} is down"
      
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.01
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High error rate on {{ $labels.service }}"
      
      - alert: HikariPoolExhausted
        expr: hikaricp_connections_active / hikaricp_connections_max > 0.8
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "HikariCP connection pool above 80%"
```

2. 创建 `monitoring/grafana-dashboard.json`:
   - JVM内存使用
   - 数据库连接池状态
   - API响应时间P99
   - 错误率监控

#### 4. 服务健康检查完善 ✅

**问题**: 缺少服务依赖顺序管理

**整改方案**:

```yaml
      show-details: always

      enabled: true

      enabled: true
```
