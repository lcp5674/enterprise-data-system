## 🔧 已修复问题清单

### P0 - 阻断性问题（已全部修复）

#### 1. 前端语法错误
- **文件**: `frontend/edams-web/src/constants/index.ts`
- **问题**: 第111行和第249行存在引号不匹配
  - `RESTORE: (id: string) => \`/api/v1/assets/${id}/restore',` → 应为 `\``
  - `REJECT: (id: string) => \`/api/v1/permission-requests/${id}/reject',` → 应为 `\``
- **影响**: 编译错误，API路径无法正确生成
- **修复**: 已修正引号为正确的反引号

#### 2. KieSession资源泄漏
- **文件**: `backend/domain/services/rule-engine-service/.../RuleEngineServiceImpl.java`
- **问题**: 每次评估都创建新的KieSession，未复用注入的会话对象
- **影响**: 内存泄漏，性能问题
- **修复**: 改用注入的qualitySession/complianceSession等会话对象

#### 3. 空指针异常风险
- **文件**: `backend/domain/services/rule-engine-service/.../RuleEngineServiceImpl.java`
- **问题**: `asset.getComplianceDetails().clear()` 未检查null
- **影响**: NullPointerException
- **修复**: 添加null检查并初始化

#### 4. 安全上下文未正确获取
- **文件**: `backend/core/edams-parent/edams-asset/.../AssetController.java`
- **问题**: `getCurrentUser()` 硬编码返回 "system"
- **影响**: 审计日志无法记录真实用户，无法实现用户级别权限控制
- **修复**: 从Spring Security Context获取当前认证用户

### P1 - 严重问题（已全部修复）

#### 5. 操作人ID解析异常
- **文件**: `backend/core/edams-parent/edams-asset/.../AssetServiceImpl.java`
- **问题**: `Long.valueOf(operator)` 假设operator是数字字符串
- **影响**: 用户名作为operator时抛出NumberFormatException
- **修复**: 添加`parseOperatorToLong()`安全解析方法
