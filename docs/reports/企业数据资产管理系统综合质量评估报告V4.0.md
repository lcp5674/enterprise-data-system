## 🔴 P0 严重问题（已全部修复）

### 问题 1: context-path 与 Controller 路径不匹配 → 所有 API 404

**严重性**: P0（所有API调用失败）

**原因**: 17个服务配置了 `context-path`，但路径与 Controller 的 `@RequestMapping` 不匹配。

**示例**:
```yaml
# application.yml (错误配置)
server:
  servlet:
    context-path: /edams-auth     # ← 错误：/edams-auth

# Controller (期望路径)
@RequestMapping("/api/v1/auth")    # ← 期望：/api/v1/auth

# 实际请求: /edams-auth/api/v1/auth → 404 Not Found
```

**影响范围**: 24个服务（13个domain + 7个discovery + 4个P3服务）

**修复方案**:
- 移除所有服务的 `context-path` 配置
- Spring Cloud Gateway 通过 Nacos `lb://` 直连服务根路径
- 所有 Controller `@RequestMapping` 以 `/api/v1` 开头，由 StripPrefix=1 处理

**涉及文件**: 34个 application.yml

**状态**: ✅ **已全部修复**（V4.0本次修复）

### 问题 3: 硬编码加密密钥

**严重性**: P0（数据泄露风险）

**位置**: `datasource-service/DatasourceConfigServiceImpl.java`
```java
// ❌ 错误：硬编码默认密钥
@Value("${datasource.encryption.key:default-aes-256-gcm-key-32bytes!}")
private String encryptionKey;

// ✅ 修复：必须从环境变量注入
@Value("${datasource.encryption.key}")  // 无默认值
private String encryptionKey;
```

**修复方案**:
- 移除默认密钥，强制要求通过环境变量注入
- K8s Secret 配置（见 `infrastructure/k8s/secrets/edams-secrets.yaml`）

**状态**: ✅ **已修复**

## 🟢 P2 建议改进

### 改进 1: JWT 密钥未加密存储
- 当前: JWT 密钥通过环境变量注入（可接受）
- 建议: 使用 Vault 或 AWS Secrets Manager

### 改进 2: 数据库密码仍为测试值
- 当前: `${DB_PASSWORD:root123456}` 在 application.yml 中使用默认值
- 建议: 所有生产环境必须从 K8s Secret 注入

### 改进 3: 无全局异常处理器
- 部分服务缺少 `@ControllerAdvice` 全局异常处理
- 建议: 统一异常处理，返回标准错误码

### 改进 4: 无接口幂等性保护
- POST 类接口（创建资产、提交审批）未实现幂等性令牌
- 建议: 添加 Idempotency-Key 请求头处理
