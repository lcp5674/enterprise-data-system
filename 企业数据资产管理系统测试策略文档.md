## 4 性能测试策略

### 4.1 性能测试类型

| 类型 | 目的 | 指标 |
|------|------|------|
| 基准测试 | 确定性能基线 | 响应时间、吞吐量 |
| 负载测试 | 验证预期负载下的性能 | TPS、响应时间 |
| 压力测试 | 确定系统极限 | 最大TPS、崩溃点 |
| 稳定性测试 | 验证长期运行稳定性 | 内存泄漏、性能衰减 |
| 峰值测试 | 验证双11等活动峰值 | 10倍负载 |

### 4.2 性能指标要求

| 指标 | 目标值 | 告警阈值 |
|------|--------|----------|
| API响应时间P50 | ≤100ms | >200ms |
| API响应时间P95 | ≤500ms | >800ms |
| API响应时间P99 | ≤1000ms | >1500ms |
| 系统吞吐量 | ≥5000 TPS | <3000 TPS |
| 并发用户数 | ≥1000 | <500 |
| CPU使用率 | ≤70% | >85% |
| 内存使用率 | ≤75% | >90% |
| 错误率 | <0.1% | >1% |

## 5 安全测试策略

| 类型 | 频率 | 工具 |
|------|------|------|
| 代码安全扫描 | 每次提交 | SonarQube |
| 依赖漏洞扫描 | 每周 | Snyk/Trivy |
| 渗透测试 | 每年2次 | 第三方 |
| API安全测试 | 每次发布 | Burp Suite |
| SQL注入测试 | 每次发布 | SQLMap |
| XSS测试 | 每次发布 | OWASP ZAP |

### 5.2 安全测试用例

| 测试项 | 测试方法 | 预期结果 |
|--------|----------|----------|
| SQL注入 | 输入 '; DROP TABLE; | 查询失败，无数据泄露 |
| XSS攻击 | 输入 <script>alert(1)</script> | 脚本不执行，已转义 |
| 越权访问 | 尝试访问无权限资源 | 拒绝访问，403响应 |
| 认证绕过 | 不带Token访问 | 拒绝访问，401响应 |
| 敏感数据泄露 | 响应中检查敏感信息 | 密码/密钥不返回 |

### 6.2 CI/CD集成

```yaml
# Jenkinsfile 测试阶段
pipeline {
  stages {
    stage('Unit Test') {
      steps {
        sh 'mvn test -Dtest=*Test'
      }
      post {
        always {
          junit 'target/surefire-reports/*.xml'
          recordIssues(tools: [java()])
        }
      }
    }
    
    stage('Integration Test') {
      steps {
        sh 'mvn verify -DskipUnitTests'
      }
    }
    
    stage('Security Scan') {
      steps {
        sh 'mvn verify -Dsecurity'
      }
    }
    
    stage('E2E Test') {
      when { branch 'main' }
      steps {
        sh 'npm run test:e2e'
      }
    }
  }
}
```
