## 1 安全架构概述

### 1.1 安全设计原则

| 原则 | 说明 |
|------|------|
| 纵深防御 | 多层安全控制，任何一层失效不影响整体安全 |
| 最小权限 | 权限最小化，只授予必要权限 |
| 默认安全 | 默认配置即为安全配置 |
| 零信任 | 持续验证，永不信任 |
| 安全内建 | 安全不是事后补救，而是内建于设计 |

## 2 身份认证设计

### 2.1 认证方式

| 认证方式 | 适用场景 | 安全等级 |
|----------|----------|----------|
| 用户名+密码 | 基础认证 | 中 |
| 手机验证码 | 忘记密码 | 中 |
| 企业微信OAuth | SSO登录 | 高 |
| 钉钉OAuth | SSO登录 | 高 |
| LDAP/AD认证 | 企业内网 | 高 |
| MFA多因素 | 敏感操作 | 极高 |

## 3 权限控制设计

### 3.1 权限模型

采用RBAC+ABAC混合权限模型：

```
RBAC (基于角色) + ABAC (基于属性) = 精细化权限控制
```

#### 角色权限矩阵

| 角色 | 资产查看 | 资产编辑 | 资产删除 | 质量检测 | 敏感操作 | 系统管理 |
|------|----------|----------|----------|----------|----------|----------|
| 普通用户 | ✓(所属) | ✗ | ✗ | ✗ | ✗ | ✗ |
| 数据Owner | ✓(拥有) | ✓(拥有) | ✗ | ✓ | ✗ | ✗ |
| 数据管理员 | ✓(全部) | ✓ | ✓(审批) | ✓ | ✓ | ✗ |
| 安全管理员 | ✓ | ✗ | ✗ | ✗ | ✓ | ✓(安全) |
| 系统管理员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

### 3.2 数据权限控制

| 维度 | 控制方式 | 实现 |
|------|----------|------|
| 部门维度 | 数据隔离 | WHERE department_id IN (用户可访问部门) |
| 业务域维度 | 标签过滤 | WHERE business_domain IN (用户可访问域) |
| 敏感级别维度 | 强度认证 | L4操作需二次验证 |
| 字段维度 | 列级权限 | SELECT时过滤未授权字段 |

### 3.3 敏感操作二次验证

以下操作需要二次验证：

| 操作类型 | 验证方式 | 示例 |
|----------|----------|------|
| 删除资产 | 手机验证码 | 删除L3+资产 |
| 导出敏感数据 | MFA | 导出客户信息表 |
| 修改敏感级别 | MFA+审批 | 降低敏感级别 |
| 权限变更 | MFA | 给其他人授权L3+ |
| 系统配置变更 | MFA+审批 | 修改安全配置 |

### 4.3 敏感字段加密

```sql
-- 敏感字段加密示例
CREATE TABLE encrypted_fields (
    id VARCHAR(32),
    
    -- 手机号：可搜索加密
    phone_encrypted BYTEA,          -- AES-256加密存储
    phone_searchable BYTEA,         -- 可搜索加密（支持=查询）
    
    -- 身份证号：确定性加密
    id_card_encrypted BYTEA,       -- AES-256加密
    
    -- 密码：单向哈希
    password_hash VARCHAR(255),     -- bcrypt哈希
    
    -- 银行账号：应用层加密
    bank_account_encrypted TEXT,    -- 应用加密存储
);
```

### 4.4 脱敏策略

| 脱敏类型 | 说明 | 适用场景 |
|----------|------|----------|
| 哈希脱敏 | 不可逆哈希 | 需要唯一性但不可见原值 |
| 截断脱敏 | 部分保留 | 手机号、身份证号 |
| 替换脱敏 | 替换为假数据 | 测试环境 |
| 掩码脱敏 | 部分字符掩码 | 银行卡号 |
| 泛化脱敏 | 精度降低 | 精确位置→城市 |
| 时间泛化 | 日期精度降低 | 精确日期→年月 |

**脱敏函数示例**：

```python
def mask_phone(phone: str) -> str:
    """手机号脱敏：138****5678"""
    if len(phone) == 11:
        return f"{phone[:3]}****{phone[-4:]}"
    return phone[:3] + "****" + phone[-4:] if len(phone) > 7 else "***"

def mask_id_card(id_card: str) -> str:
    """身份证脱敏：110101****0011"""
    if len(id_card) == 18:
        return f"{id_card[:6]}********{id_card[-4:]}"
    return id_card[:6] + "****" + id_card[-4:] if len(id_card) > 10 else "****"
```

## 6 Web安全设计

### 6.1 XSS防护

| 防护措施 | 说明 |
|----------|------|
| 输入校验 | 白名单校验，禁止特殊字符 |
| 输出编码 | HTML编码、URL编码、JS编码 |
| CSP策略 | Content-Security-Policy头 |
| HttpOnly | Cookie设置HttpOnly，防止JS读取 |
| X-XSS-Protection | 浏览器XSS过滤 |

### 6.2 CSRF防护

| 防护措施 | 说明 |
|----------|------|
| CSRF Token | 每个表单包含随机Token |
| SameSite Cookie | Cookie设置SameSite=Strict |
| Referer校验 | 校验请求来源 |
| 自定义Header | API请求需携带自定义Header |

### 6.3 SQL注入防护

| 防护措施 | 说明 |
|----------|------|
| 参数化查询 | 所有SQL使用参数化 |
| ORM框架 | 使用Hibernate/MyBatis |
| 白名单校验 | 操作类型、字段名白名单 |
| 最小权限 | 数据库账号权限最小化 |

### 6.4 API安全

| 措施 | 说明 |
|------|------|
| 签名验证 | API请求签名防篡改 |
| 限流 | 每IP/用户限流 |
| 黑名单 | IP黑名单、用户黑名单 |
| 请求大小限制 | 单次请求最大10MB |
