## 1 数据同步概述

### 1.1 数据同步目标

数据同步是实现企业数据资产管理的基础能力，目标包括：

- **元数据同步**：从各种数据源自动采集元数据，构建统一的元数据中心
- **血缘同步**：基于SQL解析、日志分析等方式构建数据血缘关系
- **质量同步**：将外部数据源的质量检测结果同步到系统
- **变更同步**：实时捕获数据源变更，保持元数据时效性

### 3.3 分片策略

| 分片策略 | 适用场景 | 实现方式 | 并行度 |
|----------|----------|----------|--------|
| 主键分片 | 有自增主键的表 | 按ID范围切分 | 高 |
| 时间分片 | 有时间字段的表 | 按时间范围切分 | 高 |
| 哈希分片 | 无明显分片字段 | 按主键哈希切分 | 中 |
| 随机分片 | 所有表 | 按记录数均分 | 中 |

```java
// 主键分片策略实现
public class PrimaryKeySplitStrategy implements SplitStrategy {
    
    @Override
    public List<Range> split(DataSourceConfig config) {
        String table = config.getTable();
        String splitPk = config.getSplitPk();
        
        // 查询最大最小值
        String minSql = String.format("SELECT MIN(%s), MAX(%s) FROM %s", 
                                       splitPk, splitPk, table);
        
        try (Connection conn = getConnection(config)) {
            Statement stmt = conn.createStatement();
            ResultSet rs = stmt.executeQuery(minSql);
            
            if (rs.next()) {
                long minId = rs.getLong(1);
                long maxId = rs.getLong(2);
                
                int parallelism = config.getParallelism();
                long step = (maxId - minId) / parallelism + 1;
                
                List<Range> ranges = new ArrayList<>();
                for (int i = 0; i < parallelism; i++) {
                    long start = minId + i * step;
                    long end = Math.min(start + step, maxId);
                    ranges.add(new Range(splitPk + " >= " + start, 
                                         splitPk + " <= " + end));
                }
                return ranges;
            }
        }
        
        return Collections.singletonList(new Range("1=1", "1=1"));
    }
}
```

### 6.2 数据异常处理

| 异常类型 | 处理策略 | 动作 |
|----------|----------|------|
| 数据类型不匹配 | 转换或跳过 | 记录异常日志 |
| 数据截断 | 截断或跳过 | 记录异常日志 |
| 唯一约束冲突 | 更新或跳过 | 取决于配置 |
| 外键约束失败 | 延迟重试 | 最多重试3次 |
| 连接超时 | 重试 | 指数退避重试 |
| 批量写入失败 | 拆分为小批次 | 减少批次大小 |
