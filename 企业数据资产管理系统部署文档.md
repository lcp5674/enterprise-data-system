## 1 部署架构概述

### 1.1 部署模式

企业数据资产管理系统支持多种部署模式：

| 部署模式 | 适用场景 | 复杂度 | 资源需求 |
|----------|----------|--------|----------|
| **开发环境** | 本地开发、单元测试 | 低 | 单机 4核8G |
| **测试环境** | 功能测试、集成测试 | 中 | 单机 8核16G |
| **预发布环境** | UAT验收、性能测试 | 中高 | 3节点集群 |
| **生产环境** | 正式运营 | 高 | K8s集群 16核32G×3+ |

## 3 Docker Compose 快速部署

### 3.1 环境准备

```bash
# 克隆项目
git clone https://github.com/enterprise/edams.git
cd enterprise-data-system

# 创建网络
docker network create edams-network
```

### 3.2 启动基础服务

```bash
# 启动所有基础中间件
cd docker
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f
```

### 3.3 服务启动顺序

```
启动顺序建议：
1. 等待基础服务健康检查通过
   - MySQL (约10秒)
   - Redis (约5秒)
   - Nacos (约30秒)

2. 启动后端微服务
   - edams-auth (认证服务 - 基础依赖)
   - edams-user (用户服务)
   - edams-permission (权限服务)
   - 其他业务服务...

3. 启动前端应用
   - edams-web (Web管理平台)
```

### 3.4 验证部署

```bash
# 检查服务健康状态
curl http://localhost:8848/nacos/v1/console/health/readiness
curl http://localhost:9200/_cluster/health
curl http://localhost:7474

# 访问控制台
# Nacos: http://localhost:8848/nacos (nacos/nacos)
# Sentinel: http://localhost:8858 (sentinel/sentinel)
# Neo4j: http://localhost:7474 (neo4j/neo4j123)
# Kafka UI: http://localhost:8090
# Prometheus: http://localhost:9090
# Grafana: http://localhost:3000 (admin/admin)
```

## 4 Kubernetes 生产部署

### 4.1 集群准备

#### 4.1.1 创建命名空间

```bash
kubectl create namespace edams-system
kubectl create namespace edams-apps
```

#### 4.1.2 配置文件

所有 Kubernetes 配置文件位于 `infrastructure/k8s/` 目录：

```
infrastructure/k8s/
├── namespaces/          # 命名空间定义
├── configs/            # ConfigMap 配置
├── storage/            # 存储类定义
├── services/           # Service 定义
├── deployments/        # Deployment 定义
├── ingress.yaml        # Ingress 规则
└── README.md           # K8s 部署说明
```

### 4.2 存储配置

#### 4.2.1 创建 StorageClass

```yaml
# infrastructure/k8s/storage/storage-class.yaml
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: edams-ssd
provisioner: cloud-provider/storage-provisioner
parameters:
  type: SSD
  zone: cn-hangzhou-a
reclaimPolicy: Retain
volumeBindingMode: WaitForFirstConsumer
```

#### 4.2.2 创建 PVC

```yaml
# infrastructure/k8s/storage/mysql-pvc.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: edams-mysql-pvc
  namespace: edams-apps
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: edams-ssd
  resources:
    requests:
      storage: 100Gi
```

### 4.3 中间件部署

#### 4.3.1 MySQL 部署

```yaml
# infrastructure/k8s/deployments/mysql-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: edams-mysql
  namespace: edams-apps
spec:
  replicas: 1
  selector:
    matchLabels:
      app: edams-mysql
  template:
    metadata:
      labels:
        app: edams-mysql
    spec:
      containers:
        - name: mysql
          image: mysql:8.0
          env:
            - name: MYSQL_ROOT_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: edams-secrets
                  key: mysql-root-password
          ports:
            - containerPort: 3306
          volumeMounts:
            - name: mysql-data
              mountPath: /var/lib/mysql
      volumes:
        - name: mysql-data
          persistentVolumeClaim:
            claimName: edams-mysql-pvc
---
apiVersion: v1
kind: Service
metadata:
  name: edams-mysql
  namespace: edams-apps
spec:
  type: ClusterIP
  ports:
    - port: 3306
      targetPort: 3306
  selector:
    app: edams-mysql
```

#### 4.3.2 Redis 部署

```yaml
# infrastructure/k8s/deployments/redis-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: edams-redis
  namespace: edams-apps
spec:
  replicas: 1
  selector:
    matchLabels:
      app: edams-redis
  template:
    metadata:
      labels:
        app: edams-redis
    spec:
      containers:
        - name: redis
          image: redis:7-alpine
          command: ["redis-server", "--requirepass", "$(REDIS_PASSWORD)"]
          ports:
            - containerPort: 6379
          env:
            - name: REDIS_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: edams-secrets
                  key: redis-password
```

#### 4.3.3 Kafka 部署

```yaml
# infrastructure/k8s/deployments/kafka-deployment.yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: edams-kafka
  namespace: edams-apps
spec:
  serviceName: edams-kafka
  replicas: 3
  selector:
    matchLabels:
      app: edams-kafka
  template:
    metadata:
      labels:
        app: edams-kafka
    spec:
      containers:
        - name: kafka
          image: bitnami/kafka:3.6
          ports:
            - containerPort: 9092
              name: kafka
          env:
            - name: KAFKA_CFG_NODE_ID
              valueFrom:
                fieldRef:
                  fieldPath: metadata.name
            - name: ALLOW_PLAINTEXT_LISTENER
              value: "yes"
```

### 4.4 微服务部署

#### 4.4.1 认证服务部署

```yaml
# infrastructure/k8s/deployments/edams-auth-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: edams-auth
  namespace: edams-apps
spec:
  replicas: 2
  selector:
    matchLabels:
      app: edams-auth
  template:
    metadata:
      labels:
        app: edams-auth
    spec:
      containers:
        - name: edams-auth
          image: edams-registry.io/edams/edams-auth:v1.0.0
          ports:
            - containerPort: 8080
          env:
            - name: SPRING_PROFILES_ACTIVE
              value: "prod"
            - name: NACOS_SERVER_ADDR
              value: "nacos.edams-system.svc.cluster.local:8848"
            - name: MYSQL_HOST
              value: "edams-mysql.edams-apps.svc.cluster.local"
            - name: REDIS_HOST
              value: "edams-redis.edams-apps.svc.cluster.local"
          resources:
            requests:
              memory: "512Mi"
              cpu: "250m"
            limits:
              memory: "1Gi"
              cpu: "500m"
          livenessProbe:
            httpGet:
              path: /actuator/health/liveness
              port: 8080
            initialDelaySeconds: 60
            periodSeconds: 10
          readinessProbe:
            httpGet:
              path: /actuator/health/readiness
              port: 8080
            initialDelaySeconds: 30
            periodSeconds: 5
---
apiVersion: v1
kind: Service
metadata:
  name: edams-auth
  namespace: edams-apps
spec:
  type: ClusterIP
  ports:
    - port: 8080
      targetPort: 8080
  selector:
    app: edams-auth
```

### 4.5 Ingress 配置

```yaml
# infrastructure/k8s/ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: edams-ingress
  namespace: edams-apps
  annotations:
    nginx.ingress.kubernetes.io/proxy-body-size: "100m"
    nginx.ingress.kubernetes.io/proxy-read-timeout: "300"
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
spec:
  ingressClassName: nginx
  rules:
    - host: edams.example.com
      http:
        paths:
          - path: /api
            pathType: Prefix
            backend:
              service:
                name: edams-gateway
                port:
                  number: 8080
          - path: /
            pathType: Prefix
            backend:
              service:
                name: edams-web
                port:
                  number: 80
```

### 4.6 部署脚本

```bash
#!/bin/bash
# infrastructure/k8s/deploy.sh

set -e

NAMESPACE="edams-apps"
REGISTRY="edams-registry.io"

echo "====== 开始部署企业数据资产管理系统 ======"

# 1. 创建命名空间
echo ">> 创建命名空间..."
kubectl apply -f namespaces/

# 2. 部署存储
echo ">> 部署存储..."
kubectl apply -f storage/

# 3. 部署配置
echo ">> 部署配置..."
kubectl apply -f configs/

# 4. 部署中间件
echo ">> 部署中间件..."
kubectl apply -f deployments/mysql-deployment.yaml
kubectl apply -f deployments/redis-deployment.yaml
kubectl apply -f deployments/kafka-deployment.yaml

# 等待中间件就绪
echo ">> 等待中间件就绪..."
kubectl wait --for=condition=available deployment/edams-mysql -n $NAMESPACE --timeout=300s
kubectl wait --for=condition=available deployment/edams-redis -n $NAMESPACE --timeout=300s

# 5. 部署微服务
echo ">> 部署微服务..."
for svc in edams-auth edams-user edams-permission edams-workflow; do
    kubectl apply -f deployments/${svc}-deployment.yaml
done

# 6. 部署Ingress
echo ">> 部署Ingress..."
kubectl apply -f ingress.yaml

echo "====== 部署完成 ======"
kubectl get pods -n $NAMESPACE
```

## 5 配置管理

### 5.1 Nacos 配置

#### 5.1.1 应用配置

```yaml
# Nacos 配置中心 - edams-auth.yaml
spring:
  application:
    name: edams-auth
  cloud:
    nacos:
      discovery:
        server-addr: ${NACOS_HOST:nacos}:8848
        namespace: ${NACOS_NAMESPACE:public}
      config:
        server-addr: ${NACOS_HOST:nacos}:8848
        file-extension: yaml
        shared-configs:
          - data-id: common.yaml
            group: DEFAULT_GROUP
            refresh: true

# 数据库配置
  datasource:
    url: jdbc:mysql://${MYSQL_HOST:localhost}:3306/${MYSQL_DB:edams}?useSSL=false&serverTimezone=Asia/Shanghai
    username: ${MYSQL_USER:root}
    password: ${MYSQL_PASSWORD:root123456}

# Redis配置
  redis:
    host: ${REDIS_HOST:localhost}
    port: 6379
    password: ${REDIS_PASSWORD:redis123}

# Kafka配置
  kafka:
    bootstrap-servers: ${KAFKA_SERVERS:localhost:9092}
```

## 6 监控与告警

### 6.1 Prometheus 配置

```yaml
# infrastructure/k8s/configs/prometheus-config.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: prometheus-config
  namespace: edams-apps

  prometheus.yml: |
    global:
      scrape_interval: 15s
      evaluation_interval: 15s
    scrape_configs:
      - job_name: 'edams-services'
        kubernetes_sd_configs:
          - role: pod
        relabel_configs:
          - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
            action: keep
            regex: true
```

### 6.2 告警规则

```yaml
# infrastructure/k8s/configs/alert-rules.yaml
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: edams-alerts
  namespace: edams-apps
spec:
  groups:
    - name: edams.rules
      rules:
        - alert: ServiceDown
          expr: up == 0
          for: 1m
          labels:
            severity: critical
          annotations:
            summary: "服务 {{ $labels.instance }} 已停止"
        - alert: HighMemoryUsage
          expr: (container_memory_usage_bytes / container_spec_memory_limit_bytes) > 0.9
          for: 5m
          labels:
            severity: warning
```

## 7 常见问题

### 7.1 Pod 无法启动

```bash
# 查看 Pod 状态
kubectl get pods -n edams-apps

# 查看详细日志
kubectl describe pod <pod-name> -n edams-apps
kubectl logs <pod-name> -n edams-apps

# 常见原因
# 1. 镜像拉取失败 - 检查镜像地址和凭证
# 2. 配置缺失 - 检查 ConfigMap 和 Secret
# 3. 资源不足 - 检查资源配额
```

### 7.2 服务无法访问

```bash
# 检查 Service
kubectl get svc -n edams-apps
kubectl describe svc <service-name> -n edams-apps

# 检查 Endpoint
kubectl get endpoints -n edams-apps

# 检查网络策略
kubectl get networkpolicy -n edams-apps
```

### 7.3 存储问题

```bash
# 检查 PVC 状态
kubectl get pvc -n edams-apps

# 检查 PV 状态
kubectl get pv

# 常见问题
# 1. PVC Pending - 检查 StorageClass 是否存在
```
